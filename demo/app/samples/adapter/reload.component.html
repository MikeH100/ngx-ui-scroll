The first Adapter API method is the reload method. It allows to reload the uiScroll which includes resetting the items Buffer and the Viewport params. The reload method has 1 optional argument: index. It specifies the index of the item to be first in the visible part of the viewport after reload.

If index argument is not set, settings.startIndex will be used as the first index. If neither index argument not startIndex setting is present, the default value 1 will be used. See also startIndex setting description.

The Adapter.reload method is safe and needs not to be protected with the Adapter.relax (though it could be protected if the UX demands). This method interrupts the uiScroll flow and synchronously terminates any pending processes on the Scroller's end. The same is true also for the Adapter.reset method.
