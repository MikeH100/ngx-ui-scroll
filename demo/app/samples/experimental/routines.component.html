Routines API allows to adjust/reimplement DOM-specific logic within the core. This can be seen as the lowest-level API and should be considered dangerous. In order to use Routines API, you need to provide Routines parameter into the ui-scroll directive. This parameter is a class that must be extended from the original Routines class distributed by the library.

This demo implements an artificial use case where we want to access and log data items before they are rendered. The Routines factory (getCustomRoutines) is needed to provide the demo component context as we want to enable/disable the logging feature.

A complete list of Routines methods can be taken from the definition of the IRoutines interface.
