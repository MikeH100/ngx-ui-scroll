Available since v1.9.0, undocumented

onBeforeClip setting allows to run some code at the moment when 1 or more items are to clip out of the viewport. The code will run synchronously. Clipping cannot be cancelled. It is like to OnDestroy Angular lifecycle hook, but it is triggered before the elements are removed from DOM.

The log in this demo shows exactly what the onBeforeClip method sends to the console. The argument of the callback is the array of items of ItemAdapter type:
